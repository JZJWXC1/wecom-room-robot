# 设计决策记录

本文件用于记录项目模块级设计思路、边界、取舍和验证方式。每完成一个模块后，追加 3-5 句话；如果涉及重大设计决策，应先在实现前给出 2-3 个方案对比，再把最终选择和原因记录到这里。

## 2026-07-04 文档协作规范增强

本次变更属于文档与协作流程模块，不改变 Agentic RAG、素材同步、发送阶段或线上运行行为。设计上把重大设计先比选、模块完成后留痕、平台限制与设计选择分开说明三条规则写入 `AGENTS.md` 的修改流程，使后续所有执行者在动手前就能看到它们。`CLAUDE.md` 只保留同等语义的硬规则摘要，并继续声明以 `AGENTS.md` 全文为准，避免两个入口产生冲突。新增 `docs/DECISIONS.md` 作为模块设计记录的固定落点，降低未来“要求追加但文件不存在”的执行摩擦。验证方式是文档结构检查和 diff 审阅；此变更不涉及微信、企业微信、飞书或阿里云平台限制，属于项目协作设计选择。
