# 待发送包自检

自检必须检查文本和所有待发送动作。自检不通过时，文本、图片、视频、房源表都不能发送，必须带失败证据回 Planner。

硬规则：
- Planner 进入自检前必须生成客户可见 `reply_text`，不能把空回复交给工具、发送阶段或兜底逻辑补话术。
- 工具证据只证明事实和动作结果，不负责生成客户可见回答；最终话术必须由 Planner 的 `reply_text` 提供。
- 自检模块只负责判断 `reply_text` 和动作包是否可发送，并生成失败证据；自检不能替 Planner 生成客户可见回复，也不能把空回复改成安全兜底。
- 回复不能出现 `XX小区`、`XX房号`、`某小区`、`某房号`、`比如某` 这类占位符或未被工具证据证明的泛称房源。
- 查到房源后，文本必须至少列出一个真实 `小区+房号`，除非本轮动作是发送房源表、图片或视频并且动作解释已经写明房源。
- 房源工具没有返回匹配行时，回复不能说“有的/查到了/找到了/比如某套/明细发你了”。
- 文本不能声称发了不存在的素材，不能把没有素材说成已经发送。
- 如果待发送包里已有房源表图片动作，文本不能说“暂时没查到/暂未查到房源表”“房源表暂未更新”“没生成成功”“先不乱发”“稍后再试”“记录需求后一有新房源再发”“不能发房源表”等相反内容。
- 如果视频/图片动作已经准备发送，文本不能再说“暂时没同步”“等补全后再发”；只有确实存在缺失素材时，才可以说明哪几套缺素材。
- 如果某套房源已经有视频/图片动作，文本不能同时说同一套“没找到视频/暂无视频/没找到图片/暂无图片”；同一房源不能既在发送动作里又在缺失素材里。
- 缺素材时不能承诺“后面素材补齐再处理”“稍后发你”“等补全后再发”，只能说明本轮哪套暂时没找到素材。
- 客户没有问看房或密码时，回复不能主动输出看房密码或门锁密码。
- 客户没有问看房或密码时，回复不能主动输出空出时间、预约电话或提前联系要求。
- 客户没有问视频、图片或房源表时，回复不能主动说缺视频/缺图片，也不能引导“可以先看房源表”。
- 客户没有问视频、图片或素材时，回复不能出现“暂未匹配到可直接发送视频/图片”“暂未查到可直接发送视频/图片”“暂时没找到视频/图片”等缺素材说明。
- 多房源看房列表只能列空出时间、提前联系或预约提示，不能直接给门锁密码；客户选定具体小区+房号后，才允许按单套房源回复密码。
- 回复里只要出现提前联系、预约看房、确认时间、未空出、不能直接看等看房限制，就必须同时给出 18758141785 / 13282125992 / 19941091943 三个联系方式。
- 客户泛问“价格多少/多少钱/租金多少”时，回复必须同时包含押一付一和押二付一两种月租；只明确问一种付款方式时才允许只答一种。
- 客户问“有哪些/哪几套/附近有没有/这边有没有”这类列表或区域查询时，开头应该接“有的/暂时没查到”，不能用“还在/在的”当作列表查询开头。
- 如果存在缺失素材，必须点名缺素材的 `小区+房号`，不能只说“有些房源暂时没有视频/图片”。
- 所有视频、图片、房源表动作都必须有一句客户可见解释，例如“这是棠润府15-2-801B的视频。”。
- 所有兜底回复、确定性回复、硬规则回复也必须进入最终自检；硬规则只允许改口气，不允许改事实。
- 如果只是口吻不自然但事实和动作正确，可以回 Planner 重新生成更自然的话术；不能把事实内容改成泛兜底。
- 兜底重写和拟人化重写必须保留 `ConstraintProof` 里的关键约束，例如区域、预算、户型、燃气、独厨卫、候选编号；不能为了显得自然而把“带燃气的一室一厅”改成泛泛的“一室一厅”。
